# ArchSmith Project Standard

Use this skill when starting a new Spring Boot project or auditing an existing one against team standards.

## When to Use
- Starting a new backend service
- Reviewing project structure compliance
- Onboarding new team members
- Auditing tech stack alignment

## What it Provides
- Module structure template (DDD + Clean Architecture)
- Code style rules (Google Java Style + Lombok conventions)
- Configuration standards (Spring profiles, arch-smith prefix)
- JDK 25 feature adoption checklist
- Testing strategy (Spock + JUnit 6 + Testcontainers)
- Deployment patterns (Docker + jlink + Leyden CDS)
